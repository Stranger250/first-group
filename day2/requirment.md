1. 多线程写入 records 为什么要加 Lock？不加锁会出现什么问题？
答：因为 Python 的列表在底层虽然部分操作是原子的，但append操作在多线程高并发下仍可能因为线程切换导致数据竞争。
不加锁会导致：数据丢失、记录覆盖、甚至引发底层数据结构损坏导致程序崩溃。加 Lock 保证了同一时刻只有一个线程能修改 records，确保数据完整性。

2. 本案例多线程能提速，纯 CPU 计算场景还能加速吗？为什么？
答：不能。因为 Python 存在全局解释器锁。本案例中模型推理使用了time.sleep()，属于 IO 密集型任务，线程在等待 IO 时会释放 GIL，从而允许多线程并发执行。
但在纯 CPU 计算场景下，线程会一直持有 GIL，导致多线程实际上退化为单线程轮流执行，不仅无法加速，还会因为频繁的线程上下文切换增加额外开销。

3. 父类抛出 NotImplementedError 是什么设计思想，作用是什么？
答：这是“模板方法模式”和“多态”的设计思想。
作用是：强制约束子类必须重写并实现predict方法。如果开发者在编写子类时忘记实现该方法，在调用时会立即抛出明确的异常，防止程序在运行时产生难以排查的静默错误，提高了代码的健壮性和可维护性。
